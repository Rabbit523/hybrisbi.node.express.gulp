BI Data

About BI Data Imports

BI data is used for score card generation and the service report dashboard
All data is coming from the BI tool
  • All Imports from BI must be in csv format and follow below templates
  • File must be selected for import to be performed, empty file fields will be ignored
  • All Imports Assume the import contains data for the year selected in the "Time Frame Configuration" Section above, all data for the year will be wiped and reimported
Incoming Incidents File:
Open Pre Configured Incoming Incidents Query
Query Recreation:
  • 1. Open the Incoming Incidents (PS) File and set the calendar month to greater than january of the year for which you want to import data, click go
  • 2. Click "Show Filters", drag the "Applic. area" field to the filters are and set the value to "CEC HCS" and include all sub nodes
  • 3. drag "key figures" into the columns section
  • 4. drag: "Cal. year/month","Calendar day","Business partner","System Number","Current Priority","IRT compliance","Applic. area initial","Applic. area" in the exact order listed here to the rows section
  • 5. Right click "Cal year/month" field, click properties>characteristic set the field "Display" to "Key"
  • 6. Right click the "Business Partner" field, click properties>characteristic set the field "Display" to "Key" and click ok
  • 7. Right click the "System Number" field, click properties>characteristic set the field "Display" to "text" and click ok
  • 8. Right click the "Applic. Area initial", field, click Hierarchy>Hierarchy Active
  • 9. Right click the "Applic. Area" field, click Hierarchy>Hierarchy Active
The query should now look like the picture below, you may click Actions>Save View to save it for later use and the click Export>Export to CSV

SPC Data

About SPC Data Imports

SPC data is used for the service report dashboard
All data is coming from the SPC tool
  • All Imports from SPC must be in excel format and follow below templates
  • File must be selected for import to be performed, empty file fields will be ignored
  • All Imports Assume the import contains data for the year selected in the "Time Frame Configuration" Section above, all data for the year will be wiped and reimported
  • All Imports: Click Personalise > this screen in the upper right, Slect the fields in to display in EXACT order that follows: Request ID, Status, Template Name, Request Name, Created On, Implementation From,Implementation to, Changed on
  • All Imports: Click "Advanced", set "CR Created On" to greater then Jan. 1st of the current year, set "Status" To exclude :ABORT, SREJ,DRAFT,set "Product Line" to "Hybris Cloud Services"
  • All Requests Import: set "Team Responsible" to blank, save this query and export
  • PDS Requests Import: set "Team Responsible" to "HCS PLATFORM DEVOPS SUPPORT", save this query and export
  • PSE Requests Import: set "Team Responsible" to "HCS PLATFORM SERVICE ENGINEERING", save this query and export
All Queries Should look as the picture below please remember which export your file corresponds to as the export from SPC will have the same name always.(it is recommended to rename the files before importing)
